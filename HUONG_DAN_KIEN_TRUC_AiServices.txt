HUONG DAN KIEN TRUC VA QUY TAC PHAT TRIEN DU AN AiServices
==============================================================

Du an: AiServices
Muc dich: Xay dung AI API noi bo cho ERP
Kieu trien khai: AI chay tap trung tren may chu noi bo, ERP chi goi AI API qua HTTP
Phien ban tai lieu: 1.0
Ngay lap: 27/05/2026


==============================================================
1. MUC TIEU KIEN TRUC
==============================================================

AiServices la he thong AI API trung tam cho ERP noi bo.

Nguyen tac tong quat:

1. ERP WPF Client KHONG goi truc tiep Ollama.
2. ERP WPF Client chi goi AiServices.Api qua HTTP.
3. AiServices.Api moi la cong trung tam dieu phoi:
   - Xac thuc token ERP.
   - Kiem tra quyen user.
   - Goi Ollama.
   - Dieu phoi Chat/RAG.
   - Dieu phoi AI Agent sau nay.
   - Ghi log/audit.
4. AiServices khong tu xay he thong dang nhap rieng.
5. AiServices phai dung lai token dang nhap tu ERP.
6. Bat ky chuc nang doc DB, doc tai lieu, gui mail, tao bao cao, agent... deu phai di qua AiServices.Api.
7. Khong viet tat ca logic vao Controller.
8. Khong hard-code user, token, role, companyId, duong dan may chu, model name trong code.
9. Nhung phan chua co thong tin that tu ERP thi de interface/placeholder sach, khong tao demo
10. Phat trien theo tung giai doan, khong nhay thang vao Agent khi nen Chat/Auth/RAG chua on dinh.

==============================================================
5. QUY TAC AUTH ERP
==============================================================

AiServices khong xay login rieng.

Nguyen tac:
1. ERP login truoc.
2. ERP co access token/session token.
3. WPF gui token len AiServices.Api bang header:

   Authorization: Bearer <ERP_ACCESS_TOKEN>

4. AiServices.Api validate token bang mot trong cac cach sau:
   - Goi ERP Auth API.
   - Decode JWT neu ERP dung JWT.
   - Kiem tra token/session trong DB neu ERP luu DB.
5. Sau khi token hop le, AiServices tao ErpUserContext:
   - CompanyId
   - EmployeeId
   - EmployeeName
   - PartId
   - PartCode
   - PartName
   - Roles
6. Tat ca logic ve sau chi dung ErpUserContext da xac thuc.
7. Khong nhan CompanyId/EmployeeId/Role tu body request.
8. Neu token sai/het han/chua cau hinh auth thi tra 401/403, khong fake user.

Khong duoc lam:
- Khong hard-code dev token.
- Khong hard-code admin user.
- Khong cho request truyen role tu body.
- Khong bo qua auth cho endpoint chat/agent/document neu co du lieu noi bo.

Endpoint co the public:
- GET /api/ai/health

Endpoint phai auth:
- POST /api/ai/chat
- POST /api/ai/documents/sync
- GET  /api/ai/documents/search
- POST /api/ai/agent/run
- POST /api/ai/agent/approve

==============================================================

RAG chua lam o giai doan dau, nhung khi lam phai theo nguyen tac sau:

1. Tai lieu goc co the nam o thu muc M:\ hoac kho noi bo.
2. Khong tra loi dua tren tai lieu neu user khong co quyen xem.
3. Moi tai lieu phai co metadata:
   - CompanyId
   - FilePath
   - FileName
   - Title
   - DepartmentCode
   - Version
   - Status
   - EffectiveDate
   - LastModified
   - FileHash
4. File bi xoa/sua cu phai mark OBSOLETE/DELETED, khong dung de tra loi moi.
5. Text phai duoc cat chunk.
6. Chunk phai co embedding.
7. Search phai lay top chunk theo vector score.
8. Ket qua tra loi phai co source/citation.
9. Neu khong du nguon thi phai noi ro khong du du lieu.
10. Khong de AI tu che quy trinh noi bo khi khong co tai lieu/source.

Cac service du kien:
- DocumentScannerService
- TextExtractorService

==============================================================
8. QUY TAC AI AGENT
==============================================================

Agent chua lam ngay, nhung phai giu kien truc de sau nay mo rong.

Cap hanh dong:

LEVEL 0 - READ ONLY
- Doc tai lieu.
- Doc DB theo quyen.
- Tao bao cao doc.
- Khong can approval rieng neu user da co quyen xem.

LEVEL 1 - DRAFT
- Tao mail nhap.
- Tao bao cao nhap.
- Tao phieu nhap.
- Bat buoc hien cho user xem truoc.

LEVEL 2 - WRITE SAFE
- Luu phieu nhap.
- Tao task.
- Cap nhat trang thai nho.
- Can user xac nhan.

LEVEL 3 - HIGH RISK
- Gui mail ra ngoai.
- Duyet/huy phieu.
- Sua/xoa du lieu DB.
- Tao but toan.
- Anh huong kho/ke toan.
- Bat buoc phe duyet ro rang va log day du.

LEVEL 4 - FORBIDDEN BY DEFAULT
- Xoa du lieu hang loat.
- Chay script khong kiem soat.
- Tu dong gui mail hang loat khong duyet.
- Tu dong sua phan quyen/user.
- Mac dinh cam neu chua co quy trinh dac biet.

Nguyen tac:
1. Agent doc du lieu truoc, hanh dong sau.
2. Tool nao duoc phep phai dang ky trong ToolRegistry.
3. Moi tool call phai log input/output/status.
4. Hanh dong nguy hiem phai co approval.
5. Agent khong duoc tu bo qua approval.
6. Agent khong duoc tu sua phan quyen.
7. Agent khong duoc tu xoa hang loat.

18. QUY TAC ERROR HANDLING
==============================================================

1. Loi thieu token/sai token: 401.
2. Co token nhung khong co quyen: 403.
3. Request sai format: 400.
4. Khong tim thay tai nguyen: 404.
5. Loi business co the du doan: 409 hoac 422.
6. Loi he thong: 500.

Khong tra stack trace cho client production.

Nen co middleware:
- ExceptionHandlingMiddleware

Response loi nen ro rang:
{
  "message": "Noi dung loi de nguoi dung hieu",
  "traceId": "..."
}

24. GHI NHO CUOI CUNG
==============================================================

AiServices la nen tang AI noi bo dai han.

Muc tieu khong phai chi chat voi model.
Muc tieu la xay mot AI API co the phat trien thanh:
- Chat noi bo.
- RAG theo tai lieu cong ty.
- Tro ly ERP.
- AI Agent doc du lieu.
- AI Agent tao de xuat.
- AI Agent thuc hien hanh dong co phe duyet.
- He thong log/audit day du.

Viec gi chua ro thi khong hard-code tam.
Hay tao interface/placeholder sach, cho den khi co thong tin that tu ERP/DB/quy trinh.

Ket thuc tai lieu.
